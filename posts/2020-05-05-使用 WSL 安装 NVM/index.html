使用 WSL 安装 NVM

May 5, 2020 edited on: Jan 30, 2023
记录

补充：WSL 下热更新存在延迟建议还是使用 Windows，参考 在 Windows 下面使用多个 NodeJS 版本

# 安装 NVM

使用 Root 或者常用的用户

# 移动至安装目录
cd /usr/local
# 新建文件夹
mkdir nvm
# 移动至src文件夹
cd src
sudo git clone https://github.com/creationix/nvm.git

编辑~/.bashrc文件

vim ~/.bashrc

在里面追加

export NVM_DIR=/usr/local/nvm
source /usr/src/nvm/nvm.sh

然后重新登录后验证下

nvm --version

正常输出就没问题

# 安装 NodeJS

查看可安装的 NodeJS 版本

nvm -ls-remote

安装稳定版本12.16.3

nvm install 12.16.3

# 安装常用的 Yarn 包

安装yarn

npm -g i yarn

然后进入项目重新安装一次项目

# 设置代理

有时候可能会用到代理

alias socks='ALL_PROXY=socks5://127.0.0.1:1080/ \
        http_proxy=http://127.0.0.1:1080/ \
        https_proxy=http://127.0.0.1:1080/ \
        HTTP_PROXY=http://127.0.0.1:1080/ \
        HTTPS_PROXY=http://127.0.0.1:1080/'
